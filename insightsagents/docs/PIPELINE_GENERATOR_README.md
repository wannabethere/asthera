# Pipeline Code Generator

A powerful tool that converts ML pipeline code using BasePipe classes into standalone, executable Python scripts. This system creates a cache-based architecture that allows DataFrames to be stored and retrieved by name, making it easy to execute complex ML pipelines independently.

## 🚀 **What It Does**

The Pipeline Code Generator takes your ML pipeline code like this:

```python
# Step 1: Data Selection
result = (
    MetricsPipe.from_dataframe("Purchase Orders Data")
    | Select(selectors=['Date', 'Region', 'Project', 'Source', 'Transactional value'])
    ).to_df()

# Step 2: Daily Aggregation
result = (
    TrendsPipe.from_dataframe(result)
    | aggregate_by_time(group_by=['Date', 'Region', 'Project', 'Source'], aggregation={'mean': 'Transactional value'})
    ).to_df()
```

And converts it into a complete, executable Python script that:

✅ **Loads data** from specified sources  
✅ **Executes the pipeline** step by step  
✅ **Manages DataFrames** using a smart cache system  
✅ **Displays results** with comprehensive analysis  
✅ **Saves outputs** to files for further use  

## 🏗️ **Architecture Overview**

### **Key Components:**

1. **`PipelineCodeGenerator`** - Main class that parses and generates code
2. **`DataFrameCache`** - Cache system for storing DataFrames by name
3. **Data Loading Functions** - Customizable functions for loading data from various sources
4. **Generated Executable** - Complete Python script that can run independently

### **How It Works:**

```
Pipeline Code → Parser → Code Generator → Executable Script
     ↓              ↓           ↓              ↓
  Your ML      Step-by-step  Cache-based   Standalone
Pipeline      Analysis      Architecture   Executable
```

## 📝 **Pipeline Code Format**

### **Required Structure:**

```python
# Step 1: Description of what this step does
result = (
    PipeClass.from_dataframe("Data Source Name")
    | operation(parameters)
    ).to_df()

# Step 2: Next step description
result = (
    NextPipe.from_dataframe(result)  # Use 'result' for chaining
    | next_operation(parameters)
    ).to_df()
```

### **Key Elements:**

1. **Step Comments**: Each step must start with `# Step N: Description`
2. **Pipe Classes**: Use your BasePipe-derived classes (MetricsPipe, TrendsPipe, etc.)
3. **Data Sources**: Specify data source names in `from_dataframe("name")`
4. **Chaining**: Use `result` as input for subsequent steps
5. **Operations**: Apply your custom operations using the pipe operator `|`

### **Example Pipeline:**

```python
# Step 1: Load and filter customer data
customers = (
    CustomerPipe.from_dataframe("Customer Database")
    | filter_by_status(status='active')
    | select_columns(['id', 'name', 'email', 'region'])
    ).to_df()

# Step 2: Analyze purchase patterns
purchase_analysis = (
    AnalyticsPipe.from_dataframe(customers)
    | analyze_purchases(group_by=['region'], metrics=['total_spent', 'order_count'])
    ).to_df()

# Step 3: Generate insights
insights = (
    InsightsPipe.from_dataframe(purchase_analysis)
    | generate_recommendations(threshold=1000)
    ).to_df()
```

## 🗄️ **Cache System**

### **How It Works:**

The generated executable includes a `DataFrameCache` class that:

```python
class DataFrameCache:
    def store(self, name: str, df: pd.DataFrame):
        """Store a DataFrame in the cache."""
        self._cache[name] = df.copy()
    
    def retrieve(self, name: str) -> pd.DataFrame:
        """Retrieve a DataFrame from the cache."""
        return self._cache[name].copy()
    
    def list_available(self) -> List[str]:
        """List all available DataFrame names."""
        return list(self._cache.keys())
```

### **Benefits:**

- **Memory Efficient**: DataFrames are stored once and reused
- **Name-Based Access**: Access DataFrames by descriptive names
- **Automatic Management**: Cache is automatically populated and managed
- **Debugging**: Easy to inspect what data is available at each step
