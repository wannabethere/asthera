# Document Planning System

This document describes the enhanced document planning system that integrates with docmodels for intelligent document retrieval, grading, and question answering.

## Overview

The Document Planning System provides a comprehensive framework for:
- **Intelligent Document Retrieval**: Uses docmodels to retrieve relevant documents from multiple sources
- **Retrieval Quality Grading**: Evaluates and grades the quality of document retrieval
- **Strategic Planning**: Creates execution plans based on question type and retrieval quality
- **Plan Execution**: Executes plans to generate comprehensive answers

## Components

### 1. Enhanced Document Planner (`enhanced_document_planner.py`)

The core planning component that:
- Retrieves documents using docmodels
- Grades retrieval quality
- Creates execution plans based on question analysis
- Determines optimal strategies

**Key Classes:**
- `DocumentRetrievalService`: Handles document retrieval and grading
- `EnhancedDocumentPlanner`: Main planning logic
- `DocumentPlan`: Represents execution plans
- `RetrievalGrade`: Grades for retrieval quality

**Retrieval Grades:**
- `EXCELLENT`: Highly relevant, comprehensive coverage
- `GOOD`: Relevant documents with some gaps
- `FAIR`: Partially relevant with significant gaps
- `POOR`: Low relevance with major gaps
- `INSUFFICIENT`: No relevant documents found

### 2. Document Plan Executor (`document_plan_executor.py`)

Executes the plans generated by the planner:
- Implements various document analysis functions
- Handles step-by-step execution
- Provides detailed execution results
- Supports different analysis strategies

**Key Functions:**
- `analyze_document_content`: Comprehensive content analysis
- `extract_relevant_information`: Targeted information extraction
- `perform_comparison`: Cross-document comparison
- `extract_structured_data`: Structured data extraction

### 3. Document Planning Service (`document_planning_service.py`)

Unified service that combines planning and execution:
- Orchestrates the entire process
- Provides high-level API
- Handles error management
- Formats responses

**Main Methods:**
- `answer_question()`: Complete question answering workflow
- `get_document_analysis()`: Analyze specific documents
- `search_documents()`: Search without full planning
- `get_planning_insights()`: Get planning recommendations

### 4. API Router (`document_planning_router.py`)

REST API endpoints for the document planning system:
- `/answer`: Answer questions using document planning
- `/analyze-documents`: Analyze specific documents
- `/search`: Search documents with quality analysis
- `/planning-insights`: Get planning recommendations

## Planning Strategies

The system supports multiple planning strategies based on question type and retrieval quality:

### 1. Comprehensive Analysis
- **Use Case**: Complex questions requiring full document analysis
- **When**: High-quality retrieval with comprehensive coverage needed
- **Steps**: Content analysis → Information extraction → Synthesis

### 2. Focused Extraction
- **Use Case**: Specific information extraction from limited documents
- **When**: Fair retrieval quality, need to focus on most relevant docs
- **Steps**: Identify relevant docs → Extract specific info → Generate focused answer

### 3. Comparative Analysis
- **Use Case**: Comparing information across multiple documents
- **When**: Multiple documents available for comparison
- **Steps**: Metadata analysis → Extract comparative data → Perform comparison → Synthesize

### 4. Structured Extraction
- **Use Case**: Extracting structured data (tables, lists, key-value pairs)
- **When**: Documents contain structured content
- **Steps**: Identify structured content → Extract data → Format as structured answer

### 5. Timeline Analysis
- **Use Case**: Time-based analysis of document content
- **When**: Questions involve temporal aspects
- **Steps**: Extract temporal data → Analyze timeline → Generate time-based answer

### 6. Metadata Analysis
- **Use Case**: Analysis focused on document metadata
- **When**: Questions about document properties or organization
- **Steps**: Analyze metadata → Extract relevant properties → Generate metadata-based answer

### 7. Content Summarization
- **Use Case**: Quick overview of document content
- **When**: General questions requiring document summaries
- **Steps**: Summarize content → Generate narrative answer

## Usage Examples

### Basic Question Answering

```python
from app.agents.nodes.docs.document_planning_service import DocumentPlanningService

# Initialize service
service = DocumentPlanningService(
    document_service=document_service,
    chroma_store=chroma_store,
    ingestion_service=ingestion_service
)

# Answer a question
response = await service.answer_question(
    question="What are the key financial metrics in the quarterly reports?",
    document_type="financial_report",
    max_documents=10
)

print(f"Answer: {response.final_answer}")
print(f"Strategy: {response.strategy}")
print(f"Confidence: {response.confidence}")
print(f"Documents found: {response.documents_found}")
```

### Document Analysis

```python
# Analyze specific documents
analysis = await service.get_document_analysis(
    document_ids=["doc1", "doc2", "doc3"],
    analysis_type="structured"
)

print(f"Analysis type: {analysis['analysis_type']}")
print(f"Documents analyzed: {analysis['documents_analyzed']}")
print(f"Results: {analysis['results']}")
```

### Document Search

## Extensibility

### Adding New Strategies

To add a new planning strategy:

1. Add strategy to `DocumentPlanningStrategy` enum
2. Implement strategy logic in `_determine_strategy()`
3. Add execution steps in `_generate_plan_steps()`
4. Implement execution functions in `DocumentPlanExecutor`

### Adding New Analysis Functions

To add new analysis functions:

1. Add function to `function_registry` in `DocumentPlanExecutor`
2. Implement the function with proper error handling
3. Add function to plan step generation logic

### Custom Retrieval Services

To add custom retrieval services:

1. Extend `DocumentRetrievalService`
2. Implement custom retrieval logic
3. Add grading criteria for new sources

## Troubleshooting

### Common Issues

1. **No Documents Found**
   - Check document availability in database
   - Verify search query specificity
   - Review document type filters

2. **Low Retrieval Quality**
   - Refine search query
   - Check document content quality
   - Adjust relevance thresholds

3. **Execution Failures**
   - Check service dependencies
   - Verify document access permissions
   - Review error logs for specific issues

4. **Poor Answer Quality**
   - Increase document retrieval limit
   - Try different planning strategies
   - Check question clarity and specificity
